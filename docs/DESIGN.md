# DESIGN

## 1. 목적
이 문서는 구현 이전에 합의해야 하는 설계 기준을 정의한다.
핵심은 "무엇을 만들까"보다 "어떤 경계와 제약 안에서 만들까"다.

## 2. 설계 우선순위
1. Correctness (정확성)
2. Consistency (일관성)
3. Reproducibility (재현성)
4. Scalability (확장성)

## 3. 설계 체크리스트
- [ ] 비즈니스 규칙이 domain에만 있는가?
- [ ] application은 흐름 제어만 담당하는가?
- [ ] infra 세부사항이 상위 레이어로 새지 않는가?
- [ ] 테스트/로그로 결과를 검증할 수 있는가?
- [ ] 실패 시 롤백 경로가 있는가?

## 4. 트레이드오프 원칙
선택지가 2개 이상이면 아래를 기록한다.
- 대안 A/B
- 장점/단점(각 1~2줄)
- 채택 이유
- 포기한 위험
