# PLANS 운영 규칙

## 1. 목적
복잡 작업을 계획-실행-검증 가능한 형태로 관리한다.

## 2. 디렉터리 구조
- `docs/exec-plans/active/`: 진행 중 계획
- `docs/exec-plans/completed/`: 완료 계획
- `docs/exec-plans/tech-debt-tracker.md`: 상시 부채 목록

## 3. 계획 문서 생성 기준
아래 중 하나라도 해당하면 계획 문서 필수:
- 하위 작업 3개 이상
- 다중 레이어/디렉터리 변경
- 보안/데이터/의존성 관련 리스크

## 4. 계획 문서 필수 섹션
- 목표/범위/비범위
- 작업 체크리스트
- 의사결정 로그
- 검증 증거
- 남은 리스크
- 롤백 전략

## 5. 완료 처리
- 체크리스트 완료 + 검증 증거 기입
- `completed/`로 이동
- 후속 과제는 tech-debt tracker에 등록
