# SECURITY

## 1. 기본 원칙
- 비밀정보는 코드/로그/커밋에 절대 포함하지 않는다.
- 인증/인가 경계 변경은 Stop & Ask 대상이다.
- 보안 관련 작업은 반드시 검증 증거를 남긴다.

## 2. 금지 사항
- 하드코딩된 API Key/Token/Password
- 민감정보 평문 로그
- 검증 우회(임시 allow-all, bypass 등)

## 3. 변경 시 필수 점검
- [ ] 변경 범위에 auth/security가 포함되는가?
- [ ] 영향 서비스/엔드포인트 목록화했는가?
- [ ] 롤백 방법이 있는가?
- [ ] 실패 시 안전한 기본값을 유지하는가?

## 4. 시크릿 관리
- 로컬: 환경변수/.env (커밋 금지)
- 저장소: secret manager/CI secret
- 문서에는 키 값이 아닌 키 이름만 기록

## 5. 사고 대응
- 노출 의심 즉시 키 폐기/재발급
- 영향 범위와 조치 내역 문서화
