# PRODUCT_SENSE

## 1. 목적
코드 구현 전에 사용자 가치와 수용 기준을 명확히 정의한다.

## 2. 기능 정의 템플릿
- 문제: 어떤 불편/리스크를 해결하는가?
- 대상 사용자: 누가 영향을 받는가?
- 기대 행동: 사용자가 무엇을 할 수 있어야 하는가?
- 비범위: 이번 작업에서 하지 않는 것

## 3. 수용 기준 (Acceptance Criteria)
각 기능은 최소 3가지 기준을 문서화한다.
1) 성공 시나리오
2) 실패/예외 시나리오
3) 관측 가능 지표(로그/메트릭/테스트)

## 4. 완료 정의 (Definition of Done)
- [ ] 수용 기준 충족
- [ ] 회귀 없음(테스트/검증)
- [ ] 문서 동기화
- [ ] 남은 리스크 명시

## 5. 운영 원칙
- “잘 될 것 같다” 금지, 검증 가능한 결과만 인정
- 구현 편의보다 사용자 영향/안정성 우선
