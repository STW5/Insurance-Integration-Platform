# RELIABILITY

## 1. 목표
변경 후 시스템이 예측 가능한 방식으로 동작하고, 실패해도 복구 가능해야 한다.

## 2. 검증 루틴
1) 사전 상태 확인 (health/log)
2) 변경 적용
3) 사후 상태 확인 (health/log/test)
4) 회귀 징후 확인

## 3. 필수 기록 항목
- 실행 명령
- 핵심 출력(요약)
- 성공/실패 판단 근거
- 미해결 리스크

## 4. 장애 대응 원칙
- 재현 불가 이슈는 우선 재현 절차부터 고정
- 임시 우회는 만료 조건과 제거 계획 동반
- 반복되는 오류는 exec-plan으로 승격

## 5. SLO 관점 체크(간소)
- 가용성: 핵심 엔드포인트 health
- 지연: 핵심 요청 시간 baseline 대비 악화 여부
- 오류: 5xx/예외 로그 증가 여부
