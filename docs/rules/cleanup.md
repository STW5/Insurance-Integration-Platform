# Cleanup & Entropy Management

에이전트는 기존 패턴을 복제한다. 드리프트를 방치하면 품질이 급격히 저하된다.

## 1. 정리 대상
- 미사용 코드/import/필드
- 주석 처리된 구 로직
- 임시 디버그 출력/테스트 전용 코드
- 장기 방치 TODO/FIXME
- 문서-코드 불일치

## 2. 정리 절차
1) 동작 보호 테스트 확인/추가
2) 삭제 우선 리팩터링
3) 변경 범위 최소화
4) Quality Gate 재검증
5) 기술부채 트래커 업데이트

## 3. 실행 시점
- 기능 구현 직후
- 동일 악취 패턴이 3곳 이상일 때
- 아키텍처/컨벤션 위반 발견 시
- 대규모 변경 후 문서 동기화 단계

## 4. 드리프트 관리
- 코드 변경량이 큰데 문서 변경이 없으면 경고
- rules/docs가 오래되면 우선 문서 정리
- “나중에 정리” 금지, 작은 단위로 즉시 상환
