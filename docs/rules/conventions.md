# Coding & Commit Conventions

## 1. 커밋/배포 정책
- 사용자 승인 전 commit/push/deploy 금지
- 금지 문구: `with claude`, `Co-Authored-By`
- 커밋 제목은 한국어 포함 권장(훅 규칙 적용)

## 2. 코드 스타일 원칙
- 작은 함수, 명확한 이름, early return 우선
- 하드코딩 상수 최소화
- catch-all Exception 남용 금지
- 민감정보 로그 금지
- 변경 전 파일 전체 읽기

## 3. 품질 원칙
- 구현 선택지가 2개 이상이면
  - 장단점 1줄씩 비교
  - 가장 단순하고 검증 가능한 방법 선택
- 테스트 가능한 설계를 우선

## 4. 환경/설정
- 시크릿은 환경변수 또는 안전 저장소 사용
- `.env` 커밋 금지
- 로컬 검증 명령을 문서화해 재현성 확보

## 5. PR 전 체크리스트
- [ ] 요구사항/비범위 확인
- [ ] 변경 파일 전수 확인
- [ ] 테스트/검증 실행
- [ ] 문서 동기화 확인
- [ ] 리스크/롤백 경로 확인
